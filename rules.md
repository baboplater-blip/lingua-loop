# 🚫 rules.md — 절대 불가침 규칙 (Invariants)

> **이 규칙은 깨면 안 된다.** 기능 추가·리팩터·배포 어떤 상황에서도 유지된다.
> 위반은 곧 학습성과 훼손·데이터 오염·프라이버시 사고·신뢰붕괴. 새 코드/콘텐츠는 항상 이 규칙을 통과한다.
> 검증 방법은 [test.md](test.md). 이 문서는 방향이 근본적으로 바뀌지 않는 한 수정하지 않는다.

## A. 학습 진실성 (절대 — 이 프로젝트의 존재 이유)

1. **성과가 진실, 참여는 아니다** — 모든 알고리즘·UI·실험의 최적화 목표는 **실측 학습성과**(TTM·Gain Score·리텐션)다. 체류시간·세션수·스트릭을 **최적화 대상으로 삼지 않는다**. 가드레일 지표는 항상 학습성과.
2. **간격반복은 기억과학을 따른다** — 복습 스케줄은 검증된 모델(FSRS 등)로 결정. "더 자주 보여주면 참여 늘어" 식의 비과학적 강제 노출 금지([`fsrs-spaced-repetition`](.claude/skills/fsrs-spaced-repetition/SKILL.md)).
3. **난이도·능력은 데이터로 추정, 임의값 금지** — 문항 난이도·학습자 능력은 반응 데이터로 캘리브레이션(IRT/ELO)한다. 하드코딩 난이도는 **초기 시드**로만 허용하며 캘리브레이션되면 교체([`irt-calibration`](.claude/skills/irt-calibration/SKILL.md)).
4. **콘텐츠 정확성 게이트** — 생성·기여 콘텐츠는 언어적 정확성·정답 유효성·오답보기 타당성·문화적 안전성 게이트를 통과해야 학습자에게 노출된다([`content-generation-quality-gate`](.claude/skills/content-generation-quality-gate/SKILL.md)). **미검증 콘텐츠를 학습 경로에 넣지 않는다.**

## B. 데이터·프라이버시 (절대)

5. **이벤트 로그는 append-only** — 학습 이벤트는 불변 로그에 추가만. 학습자 상태는 이벤트에서 **파생**(재현 가능). 상태 직접 덮어쓰기로 이력 파괴 금지([`telemetry-event-schema`](.claude/skills/telemetry-event-schema/SKILL.md)).
6. **학습자 데이터는 학습자 소유** — 언제든 전체 내보내기·영구 삭제 가능(삭제는 확인 강제). 데이터 이동성 보장([`privacy-consent-open-data`](.claude/skills/privacy-consent-open-data/SKILL.md)).
7. **동의 없는 개인식별 데이터 개방 금지** — 개방 데이터셋은 **동의된 범위 + 익명화(재식별 불가)** 만. PII 를 커밋·로그·공개 데이터에 남기지 않는다.
8. **수집 최소화** — 학습 개선에 필요한 데이터만 수집. 목적 외 수집·제3자 판매 금지.

## C. 윤리·다크패턴 금지 (절대)

9. **다크패턴 금지** — 손실공포(스트릭 협박), 인위적 조바심, 무한스크롤, 죄책감 유발 알림으로 참여를 강제하지 않는다. 동기 설계는 **학습 성과에 봉사** 할 때만 정당([`motivation-gamification`] 은 이 규칙 하에 동작).
10. **투명성** — 학습자는 자신이 왜 이 항목을 지금 보는지(복습 이유·난이도·진척)를 알 수 있어야 한다. 블랙박스 강요 금지.

## D. 아키텍처·개방 (절대)

11. **코어는 언어 무관** — 코어 엔진에 특정 언어 규칙을 하드코딩하지 않는다. 언어별 지식은 **언어팩(데이터/플러그인)** 으로만 주입([`language-pack-format`](.claude/skills/language-pack-format/SKILL.md)).
12. **모델은 pluggable** — AI 튜터·생성·ASR/TTS 는 **인터페이스 뒤에** 둔다. 특정 상용 API 에 코어를 종속시키지 않는다(교체·자가호스팅 가능).
13. **자가호스팅 가능** — 폐쇄형 락인 금지. 외부 서비스 없이도 최소 기능이 도는 경로를 항상 유지.
14. **저작권·라이선스 준수** — 기여·수집·생성 콘텐츠는 출처·라이선스가 명확해야 한다. 표절·무단복제 콘텐츠 금지.

## E. 개발 프로세스 (절대)

15. **게이트 그린 = 완료 조건** — 기능은 **검증 작성 → 게이트 편입 → 게이트 그린** 후에만 "완료"·커밋([test.md](test.md)).
16. **무회귀·하위호환** — 신규 기능은 기존 검증을 깨지 않는다. 스키마 변경은 마이그레이션 + 하위호환(기존 이벤트/콘텐츠 유효).
17. **효능 우선 검증** — 학습 관련 기능은 "동작한다"에 그치지 않고 **"성과에 이롭다(또는 최소한 해롭지 않다)"** 를 데이터/실험으로 확인([`ab-experiment-framework`](.claude/skills/ab-experiment-framework/SKILL.md)).
18. **배포는 명시 승인 시에만** — origin push·프로덕션 배포는 사용자가 "커밋 배포/배포해/푸시해" 등 **명시 승인** 했을 때만.
19. **사용자 영역 침범 금지** — LLM/ASR/TTS 키·유료 크레딧·초기 언어팩 시드는 **사용자/커뮤니티 제공**. 임의 생성·가짜 값 주입 금지.
20. **문서 동기화** — 기능 변경 시 CLAUDE.md 변경이력 + [status.md](status.md) + 메모리를 갱신.
21. **사용자용 문서는 한국어** — 모든 보고서·가이드·이 문서 체계는 한국어(코드 내부 식별자·공개 OSS 문서는 영어 병행 가능).

## F. 이 문서 체계 (절대)

22. **5문서 유지** — goal·plan·status·rules·test.md 는 항상 존재하고 상호 링크된다([`verification-gate`](.claude/skills/verification-gate/SKILL.md) 가 게이트에서 검증).
23. **rules.md 는 함부로 약화하지 않는다** — 규칙 삭제/완화는 명시적 사유와 사용자 합의가 있을 때만. 특히 A·C(학습 진실성·다크패턴 금지)는 이 프로젝트의 정체성이다.
