# 🎯 goal.md — LinguaLoop 목표

> 이 문서는 **"무엇을, 왜"** 를 정의한다. *어떻게* 는 [plan.md](plan.md), *지금 어디* 는 [status.md](status.md), *깨면 안 되는 것* 은 [rules.md](rules.md), *끝났는지 확인* 은 [test.md](test.md).
> 변경은 신중히 — 방향이 근본적으로 바뀔 때만. 일상 진행 기록은 status.md 에 한다.

## 1. 비전 (North Star)

**데이터로 끝없이 진화하는, 인간을 0에서 원어민 마스터리까지 끌어올리는 오픈소스 언어 마스터리 엔진.**

LinguaLoop 은 단순 문제풀이 앱이 아니다. **학습자를 실제로 원어민 수준까지 성장시키는 것** 이 유일한 목적이며, 그것을 위해 **지금까지 있었거나 없었던 모든 효과적인 방법을 총동원** 한다 — 이해가능한 입력(comprehensible input), 간격반복(SRS), 산출·교정(output), AI 튜터 몰입 대화, 발음 섀도잉, 커뮤니티 콘텐츠. 그리고 학습에서 나온 **데이터를 수집해 스스로를 개선** 하는 폐루프(closed loop)로 **끝없이 진화** 한다. 특정 언어쌍에 종속되지 않는 **다국어 범용 엔진** 이며, 누구나 자가호스팅·기여할 수 있는 **오픈소스** 다.

성공의 정의: **실제 학습자가 이 도구만으로 목표 언어를 원어민 수준까지 도달** 하고, 그 학습 데이터가 엔진을 **다음 학습자에게 더 잘 가르치도록** 만든다.

## 2. 세 개의 축 (모든 설계가 여기서 출발)

1. **마스터리 (Mastery, 결과)** — 재미·참여가 아니라 **실측된 언어 능력 향상** 이 목표. 지표는 "몇 분 했나"가 아니라 "얼마나 빨리 다음 능력 단계에 도달했나(Time-to-Mastery)".
2. **진화 (Evolution, 폐루프)** — 모든 상호작용은 데이터가 된다. 그 데이터는 콘텐츠 난이도·간격반복 파라미터·커리큘럼 경로·교수법을 **자동으로 재보정** 한다. 도구는 쓸수록 좋아진다.
3. **개방·범용 (Open & Universal)** — 언어 무관 코어 + 데이터로 주입되는 언어팩. 콘텐츠·익명 데이터·모델·코드가 열려 있고 커뮤니티가 함께 진화시킨다.

## 3. 북스타 지표 (성공 측정)

| 지표 | 정의 | 지향 |
|---|---|---|
| **TTM** | Time-to-Mastery (한 능력요소 A1→숙달까지 걸린 학습시간) | 코호트 대비 지속 단축 |
| **Retention Curve** | 학습 후 T일 뒤 회상 정확도(망각곡선) | 상향(간격반복 최적화 효과) |
| **Gain Score** | 배치평가 → 재평가 능력 상승폭(효과크기 d) | 유의미(d ≥ 0.4)·상승 |
| **Coverage→Fluency** | CEFR A1→C2→원어민 경로 완주율 | 이탈 지점 축소 |
| **Content Health** | 캘리브레이션된 문항 비율·품질 게이트 통과율 | 상향 |
| **Loop Velocity** | 진화 1사이클(데이터→개선→검증→배포) 소요·개선폭 | 단축·양(+) |

> 지표는 관측 대시보드(진화 엔진 산출)로 본다. **참여도(세션수·스트릭)는 보조지표일 뿐, 절대 북스타가 아니다**([rules.md](rules.md) C. 다크패턴 금지).

## 4. 제품 목표

- **마스터리 스택**: 입력·간격반복·산출·튜터·발음·몰입·동기를 하나의 개인화 학습 경로로 통합([[sla-pedagogue]], [[curriculum-designer]]).
- **끝없는 진화 루프**: 수집 데이터 → 문항 캘리브레이션(IRT) → 스케줄러 튜닝(FSRS) → 콘텐츠 자동 생성·보정 → 커리큘럼 최적화 → A/B 실험 → 배포 → 측정([[evolution-loop-protocol]]).
- **다국어 범용**: 언어팩(데이터)만 추가하면 새 목표어/모국어 지원. 코어는 언어 무관([[language-pack-format]]).
- **웹앱 + 데이터 백엔드**: 브라우저 학습 UI(읽기·듣기·말하기·쓰기·튜터·복습) + 이벤트 로그·학습자 모델·진화 파이프라인 백엔드.
- **AI 튜터**: 목표어 몰입 대화·즉시 교정. 모델은 교체 가능(pluggable — Claude/로컬/오픈모델)([[ai-tutor-protocol]]).
- **커뮤니티 진화**: 콘텐츠·언어팩을 기여·검증·랭킹하는 개방형 풀([[community-contribution-workflow]]).

## 5. 성공 기준 (Definition of Success)

1. ✅ **엔진 완결** — 마스터리 스택·진화 루프·다국어 코어가 동작하고 검증 게이트 그린([test.md](test.md)).
2. ✅ **효능 입증** — 배치→재평가 Gain Score 가 유의미(도구가 실제로 가르친다는 증거).
3. ✅ **자기개선 증명** — 진화 루프 1사이클이 관측 가능한 지표 개선을 만든다(캘리브레이션·리텐션·TTM).
4. ⏳ **개방 배포** — 라이선스·자가호스팅·데이터/모델 카드·기여 가이드로 오픈소스 공개.
5. ⏳ **실연동·실콘텐츠** — LLM/ASR/TTS 키, 초기 언어팩 시드는 사용자·커뮤니티 제공 영역.

> 1·2·3 은 코드·데이터로 달성한다. 4 는 OSS 스튜어드십. 5 는 사용자/커뮤니티 입력.

## 6. 비목표 (Non-Goals)

- **참여도 극대화(중독) 설계 금지** — 학습 성과가 목표이지 체류시간이 아니다.
- **한 언어쌍 전용 앱이 아님** — 범용 엔진이 정체성. 언어별 특수화는 언어팩(데이터)으로만.
- **폐쇄형 SaaS 락인 금지** — 자가호스팅 가능해야 하고 데이터는 사용자 소유·이동 가능.
- **네이티브 앱 스토어 우선 아님** — 웹앱 우선(추후 PWA/래핑은 열어둠).
- **표절·저작권 침해 콘텐츠 금지** — 생성/기여 콘텐츠는 출처·라이선스 검증([rules.md](rules.md)).
