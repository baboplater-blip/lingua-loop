# FSRS 간격반복 (기억 SSOT)

목표: **최소 복습으로 최대 리텐션**. 참여도가 아니라 장기기억이 목표(규칙 1·2).

## 상태(3요소)

- **Stability (S)** — 기억이 얼마나 오래 가는가(일). 성공 복습마다 증가.
- **Difficulty (D)** — 항목의 내재적 어려움(1~10).
- **Retrievability (R)** — 지금 회상 성공확률. `R = (1 + t/(9·S))^-1` (경과 t 일).

## 스케줄 계산

1. 목표 리텐션 `R_target`(기본 0.9, 학습자 조절 가능).
2. 다음 간격 `I = S · (1/R_target − 1) · 9` (R 이 R_target 로 떨어지는 시점).
3. 복습 결과 grade(again/hard/good/easy)로 S·D 갱신(FSRS 갱신식, w 파라미터 벡터).
4. **인출연습 우선** — 재노출보다 능동 인출(테스트 효과).

## 복습 큐 구성

- **만기(due)** 항목 우선(dueAt ≤ now).
- **신규** 항목 도입 속도 = 학습자 부하 설정 + 성과(과부하 시 감속).
- **인터리빙** — 유사 KC 섞어 변별학습 강화(블록 반복 금지).
- **투명성**(규칙 10) — 각 항목에 "왜 지금"(만기·약점) 노출.

## 학습자별 최적화

- 개인 복습 로그(`review.done` 이벤트)로 **w 파라미터 재적합**(`/calibrate`·진화 루프).
- 개인 망각곡선 반영. 표본 부족 시 전역 기본 파라미터로 폴백.
- 최적화 목표는 **예측 리텐션 오차 최소화**, 복습량 최소화 — 노출 늘리기 아님(규칙 2).

## 불변 규칙

1. **검증된 모델만**(규칙 2) — 비과학적 강제노출·"자주 보여주기"식 스케줄 금지.
2. **결정성** — 시드 고정으로 검증 가능(test.md §4). FSRS 는 결정적.
3. **효능 검증**(규칙 17) — 시뮬레이션 학습자로 스케줄러가 리텐션을 개선함을 증명(test.md §6).
