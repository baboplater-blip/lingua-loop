# 프라이버시·동의·개방데이터 (거버넌스 SSOT)

LinguaLoop 은 데이터로 진화하고 그 데이터를 개방하려 한다. 이 스킬은 그것이 **학습자를 해치지 않게** 강제한다. 충돌 시 **프라이버시가 이긴다**.

## 동의 계층(규칙 6)

| 계층 | 허용 | 기본 |
|---|---|---|
| `learn` | 내 학습 진행·복습에 사용 | ✅(필수 최소) |
| `learn+improve` | 익명 집계로 엔진 개선(캘리브레이션·최적화) | 옵트인 |
| `learn+improve+open` | 익명화 후 개방 데이터셋 기여 | 명시 옵트인 |

- 철회 가능. 철회 시 해당 범위 데이터 사용·개방 중단.
- 이벤트마다 동의 태그([`telemetry-event-schema`]).

## 익명화·개방 게이트(규칙 7)

개방 데이터셋 릴리스 전 **모두** 통과:
1. **동의 필터** — `open` 동의 이벤트만.
2. **PII 제거·분리** — 식별자 가명화, 자유텍스트 스크럽.
3. **재식별 방지** — k-익명성/집계/노이즈(차분프라이버시 고려). 준식별자 조합 점검.
4. **레드팀** — 재식별 시도가 실패함을 증명(test.md 프라이버시 레드팀). 실패 시 릴리스 반려.
5. **데이터 카드** — 무엇을·어떻게·한계·라이선스([`oss-release-standards`]).

## 데이터 소유권(규칙 6)

- **내보내기** — `/api/account/export`: 전체 이벤트+상태, 기계가독.
- **삭제** — `/api/account/delete`: 영구 파기(확인 강제), 파생·백업 포함.

## 수집 최소화(규칙 8)

- 새 이벤트 필드는 "학습개선에 필수인가" 심사(`learner-data-privacy`). 목적 외 수집·제3자 판매 금지.
- 음성·자유텍스트는 특히 민감 — 특징만 저장 지향([`pronunciation-scoring`]).

## 불변 규칙

1. **동의 없는 개방 금지**(규칙 7).
2. **"익명화했다" 주장만으론 불충분** — 재식별 테스트로 증명.
3. **프라이버시 우선** — 편의·데이터 욕심에 양보 금지.
