# 검증 게이트 구조 (SSOT)

[test.md](../../../test.md) 의 완료 정의를 집행하는 단일 판정자. `/gate` 가 실행한다(qa-verification-engineer).

## 오케스트레이션

```
/gate → 빌드 → 클린 스테이트(테스트 데이터 초기화) → 전 카테고리 순차 검증 → 일괄 판정
        전부 PASS → ✅ 통과 / 하나라도 FAIL → ❌ + 실패 로그
```

- **클린 스테이트** — 매 실행 데이터 초기화(누적 cruft 레이스 방지).
- **결정성** — 확률요소(FSRS·IRT·실험 배정)는 **시드 고정**·디버그 훅(`seedResponses`, `forceInterval`, `forceTheta`).
- **분류** — 브라우저 필요(E2E) vs API/유닛. 배열 등록.

## 검증 카테고리(test.md §3 정합)

콘텐츠·스키마 / SRS / 심리측정 / 텔레메트리·학습자모델 / 진화루프 / AI튜터 / 발음 / **효능** / 프라이버시·개방 / 커뮤니티 / 다국어·접근성 / **문서 체계**.

## 필수 검증 스타일

1. **양방향 단언**(거짓양성 방지) — 예: 캘리브레이션 후 쉬운 문항 정답률↑·어려운↓; 미학습 KC 는 복습 큐에 없음.
2. **불변식 = 실패조건** — append-only 위반·PII 누출·미검증 콘텐츠 노출·라이선스 누락 시 **반드시 FAIL**.
3. **다크패턴 안티테스트**(규칙 9) — 손실공포·강제알림·죄책감 카피가 UI 에 **없음** 을 단언(음성 테스트).
4. **프라이버시 레드팀** — 개방셋 재식별 시도 실패 단언.
5. **효능 스모크**(test.md §6) — 시뮬 학습자로 스케줄러 리텐션 개선·캘리브레이션 수렴·불량문항 차단.

## 문서 자기검증(verify-docs)

- goal·plan·status·rules·test.md **5문서 존재**(규칙 22).
- 필수 섹션/키워드(rules 의 학습진실성·프라이버시·다크패턴; goal 의 북스타).
- status 에 게이트·Phase 현황 라인. 상호 링크 무결.
- 깨지면 게이트 차단.
