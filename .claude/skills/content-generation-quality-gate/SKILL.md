# 콘텐츠 생성·품질 게이트 (SSOT)

생성/기여 콘텐츠는 이 게이트를 통과해야 `draft → verified` 로 승격되어 학습자에게 노출된다(규칙 4).

## 생성 (필요분만)

1. **격차 지정** — 진화 루프가 지목한 KC×레벨×스킬×유형([`evolution-loop-protocol`]). 무작정 양산 금지.
2. **등급 통제 생성** — 목표 CEFR 레벨 어휘·문장복잡도(i+1). 모델은 어댑터 뒤(규칙 12), 프롬프트 버전관리.
3. **스키마 준수** — [`content-item-schema`] 필드 완비(KC 태그·출처=generated·라이선스).

## 품질 게이트(모두 통과해야 verified)

| 검사 | 내용 | 실패 처리 |
|---|---|---|
| **언어 정확성** | 문법·철자·자연스러움(원어민성) | 재생성/폐기 |
| **정답 유효성** | 정답이 유일·명확히 정답 | 폐기 |
| **오답 타당성** | 오답보기가 그럴듯하되 확실히 오답, 흔한 오류 반영 | 재설계 |
| **등급 적합** | 목표 레벨 이탈 없음 | 재등급/재생성 |
| **문화적 안전** | 편향·모욕·부적절·정치민감 회피 | 폐기 |
| **중복** | 기존 아이템과 의미중복 아님 | 병합/폐기 |
| **라이선스**(규칙 14) | 저작권 원문 복제 아님, 라이선스 명시 | 거부 |

## 검증 방법

- **자동** — 언어검사·정답 파서·중복 임베딩·금칙어·레벨 분석기.
- **모델 교차검토** — 생성모델 ≠ 검토모델(자기채점 편향 회피).
- **인간/커뮤니티** — 고위험·저확신은 동료검증 큐로([`community-contribution-workflow`]).
- **사후 캘리브레이션** — 실제 난이도는 노출 후 반응으로([`irt-calibration`], 규칙 3).

## 불변 규칙

1. **미검증 노출 금지**(규칙 4) — verified 미만은 경로에 없음.
2. **효능 안티테스트**(test.md §6) — 주입한 오류 문항이 게이트에서 실제로 걸리는지 검증.
3. **AI 라벨·출처** — 생성물은 출처=AI 로 투명 표기.
