# ✅ test.md — 검증 구조 (Verification)

> **"제대로 끝났는가"를 확인하는 단일 기준.** 기능은 여기 정의된 검증을 통과해야 "완료"다.
> 규칙은 [rules.md](rules.md)(규칙 15: 게이트 그린 = 완료 조건). 게이트 구조 스펙은 [`verification-gate`](.claude/skills/verification-gate/SKILL.md).

## 1. 완료의 정의 (Definition of Done)

한 기능이 "완료"이려면 **모두** 만족:

1. 구현이 [rules.md](rules.md) 불변식을 위반하지 않는다.
2. **검증 스크립트**가 작성되어 동작·무회귀를 증명한다.
3. 그 검증이 **릴리스 게이트**에 편입된다.
4. **게이트가 그린**(전 항목 PASS)이다.
5. **학습 관련 기능은 효능 증거** 를 동반한다(규칙 17 — 성과에 이롭다/해롭지 않다).
6. 문서 동기화(CLAUDE.md 변경이력 + [status.md](status.md) + 메모리).

## 2. 릴리스 게이트 (단일 판정자)

`/gate` 커맨드 → 게이트 오케스트레이터가 빌드 후 전 검증을 순차 실행, 통과/실패 일괄 판정.
- **클린 스테이트**: 매 실행 시작 시 테스트 데이터 초기화(누적 cruft 로 인한 레이스 방지).
- **결정성**: 타이밍 의존 대신 시드·디버그 훅(`forceInterval`, `seedResponses` 등) 사용.
- **결과**: 전부 PASS → "✅ 게이트 통과", 하나라도 FAIL → "❌ 미통과 + 실패 로그".

## 4. 검증 작성 규칙

- **양방향 단언(거짓양성 방지)**: 예 — 캘리브레이션 후 쉬운 문항 정답률↑ / 어려운 문항↓, 미학습 KC 는 복습 큐에 없음.
- **효능 검증은 인과에 조심**: 관측만으로 "가르쳤다" 단정 금지. **사전등록 A/B 또는 사전-사후 통제** 로 확인([`ab-experiment-framework`]). 표본·검정력 명시.
- **다크패턴 안티테스트**(규칙 9): 스트릭 상실 협박 문구·강제 알림·손실공포 카피가 UI 에 없음을 단언하는 **음성 테스트** 포함.
- **프라이버시 레드팀**: 개방 데이터셋에서 개인 재식별 시도가 실패함을 단언.
- **결정적 시드**: 확률적(FSRS·IRT·가챠 없음) 요소는 시드 고정으로 재현.

## 5. 문서 체계 자기 검증 (verify-docs)

게이트에서 [`verification-gate`] 가:
- goal·plan·status·rules·test.md **5문서 존재**.
- 각 문서 **필수 섹션/키워드**(rules 의 학습진실성·프라이버시·다크패턴 규칙, goal 의 북스타 지표 등).
- status.md 에 **게이트 상태·Phase 현황** 라인 존재.
- 상호 링크 무결성.

→ 문서 체계가 비거나 깨지면 게이트가 막는다([rules.md](rules.md) 규칙 22).

## 6. 효능 스모크 (학습 기능 최소 기준)

학습에 영향 주는 기능은 최소한:
- 시뮬레이션 학습자(모의 반응 모델)로 **스케줄러가 리텐션을 개선** 하는지.
- **캘리브레이션이 참 파라미터로 수렴** 하는지(합성 데이터 회수 테스트).
- **콘텐츠 품질 게이트가 불량 문항을 실제로 거르는지**(주입된 오류 문항이 차단됨).
