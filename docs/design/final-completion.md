## 1. "최종"의 정의 (Definition of Final)

이 프로젝트의 정체성은 *끝없이 진화하는 엔진*이므로 "끝"은 개발 중단이 아니다. **최종 = v1.0.0**을 다음 상태로 정의한다:

> **사람이 없어도 폐루프가 안전하게 돌고(무인 운영), 도구가 가르친다는 증거 파이프라인이 갖춰졌고(효능), 적대적 검증을 통과했으며(품질·보안), 커뮤니티가 이어받을 수 있는(개방) 상태.**

v1.0.0 완료 조건(전부 충족, goal §5와 정렬):

1. **디버깅 완결** — §2 결함 사냥 4라운드에서 발견된 CONFIRMED 결함 전부 "수정 + 회귀 테스트 + 게이트 편입"(규칙 15·16). 마지막 라운드에서 신규 결함 0.
2. **E2E 완결** — 7언어 × 전 기능 라이브 매트릭스(§2 R2) 그린.
3. **보안 완결** — security-auditor 최종 감사 통과(§2 R4).
4. **효능 완결** — 사전(배치)→학습→사후(재평가) **Gain Score(효과크기 d)** 산출 파이프라인이 `/efficacy`에 편입되고, 합성 코호트로 "측정 가능함"이 검증됨(규칙 17·19 — 실학습자 실증은 커뮤니티 영역).
5. **운영 완결** — `evolve:all`·`dataset:export`·효능 스냅샷의 정례 스케줄이 실동작 검증되고 롤백 리허설 1회 완료(OPERATING.md 준수).
6. **백로그 완결** — 이슈 #2·#3 구현 닫힘, #1은 기여자용 스캐폴드 가이드로 응답.
7. **릴리스 완결** — v1.0.0 컷(semver·CHANGELOG·데이터/모델 카드 갱신·릴리스 노트 한/영). **push·공개는 '배포해' 명시 승인 시에만**(규칙 18).

## 2. 1부 — 디버깅 설계: 결함 사냥 4라운드

각 라운드의 산출은 반드시 **발견 → 최소 재현 → 수정 → 회귀 테스트 작성 → 게이트 편입 → 게이트 그린**으로만 닫는다(규칙 15). 재현 불가·오탐은 기각 기록만 남긴다.

### R1. 정적·적대적 코드 리뷰 (알고리즘·경계)
- **대상**: `packages/core`의 수리 모듈(FSRS·IRT/ELO·CAT θ 추정·BKT 숙달·efficacy 집계·open-dataset k-익명성), `packages/server` 핸들러의 입력 검증·에러 경로, `packages/adapters` 7개 튜터의 교정 규칙(오탐/미탐 경계).
- **방법**: `/code-review`(high) + code-reviewer 에이전트 병렬 리뷰. 관점: 수치 안정성(0 나눗셈·클램프·NaN 전파), 경계값(빈 이벤트·단일 표본·미지원 언어), append-only 위반 가능 경로, 다국어 정규화(유니코드·RTL·결합문자).
- **판정**: 발견 각각을 재현 테스트로 확정한 것만 결함으로 인정(적대적 검증 — 반박 우선).

### R2. 라이브 E2E 매트릭스 (실서버)
- **매트릭스**: 7언어(en·es·zh·ar·sw·ja·hi) × {CAT 배치·학습 4유형·읽기(객관식+주관식+산출)·튜터 7시나리오·발음(지각/섀도잉)·복습 큐·인증서/프로필 내보내기·계정 삭제}.
- **추가 축**: 화면 언어 5종(ar RTL 방향 전환)·CJK 클릭 사전·데바나가리 normWord·ops.html 대시보드.
- **방법**: 실서버(`npm run serve`) 기동 후 HTTP 스모크 스크립트로 전수 순회. 기존 라이브 스모크들을 하나의 재실행 가능한 매트릭스 스크립트로 통합해 게이트 편입 여부 검토(느리면 게이트 밖 `npm run e2e`로 분리 유지).

### R3. 내결함·데이터 무결
- 프로세스 강제 종료 → 재시작 복원(JSONL·SQLite 각각), 손상 이벤트 행 1줄 주입 시 서버 내성, eventId 연속성, 동시 요청 하 append 순서, 삭제(규칙 6) 후 파생 상태 재현성, JSONL↔SQLite **계약 동등성 테스트**(같은 이벤트 시퀀스 → 같은 파생 상태).
- `node:sqlite` experimental 경고: Node 버전 핀(README `>=24` 유지) + OPERATING.md에 경고 무해 명시.

### R4. 보안 최종 감사
- security-auditor: 튜터·기여 콘텐츠 경유 프롬프트/콘텐츠 인젝션 재점검, 정적 서버 경로 탐색(STATIC 화이트리스트 우회 시도), 기여 어뷰즈(신뢰가중 우회·자가 검토), 개방 데이터셋 재식별 레드팀 재실행, 시크릿·PII 커밋 스캔.
- 산출: 통과 보고서 또는 결함 → R1과 동일한 닫힘 절차.

## 4. 불변 제약 (이 설계가 지키는 것)

- **규칙 1**: 어떤 수정·지표도 학습성과 최적화 목표를 바꾸지 않는다. Gain Score는 북스타 편입, 세션수/스트릭은 계속 미노출.
- **규칙 4**: M2의 태깅·주제 확장은 "실제 포함된 문법만 정직 태깅"·게이트 통과분만 노출.
- **규칙 15·16**: 모든 결함 수정은 회귀 테스트와 함께만. 게이트 pass 수는 단조 증가.
- **규칙 18·19**: push/배포/실 LLM·ASR 키는 사용자 영역. v1.0.0도 dry-run까지가 자율 범위.
- **규칙 20·21**: 매 마일스톤마다 status.md·CLAUDE.md·메모리 동기화, 문서는 한국어.
