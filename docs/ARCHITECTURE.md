## 2. 핵심 설계 원칙 (rules.md 에서 파생)

- **이벤트 소싱** — 학습 이벤트는 불변 로그. 학습자 상태(숙달·망각곡선·능력)는 로그에서 파생·재계산 가능(규칙 5).
- **언어 무관 코어 + 언어팩** — 코어는 KC 그래프·스케줄러·평가만 안다. 언어별 지식(정서법·음운·토큰화·형태)은 언어팩이 주입(규칙 11, [`language-pack-format`]).
- **어댑터 경계** — LLM/ASR/TTS 는 어댑터 인터페이스 뒤. 기본은 자가호스팅 가능 경로 유지, 상용 API 는 교체형(규칙 12·13).
- **읽기/쓰기 분리** — 학습 경로 결정(읽기 무거움)과 이벤트 수집(쓰기 무거움)의 부하 특성이 다름 → 스토어/캐시 전략 분리 고려.

## 3. 열린 결정 → ✅ 확정 (2026-07-05, [design/architecture.md](design/architecture.md))

| 결정 | 확정 |
|---|---|
| 언어·런타임 | **TypeScript 무설치 실행**(Node 24 타입 스트리핑) + `node:test` — 제로 의존성, 자가호스팅(규칙 13) |
| 웹 프레임워크 | Next.js(App Router), 정적 export 아님 — 다음 슬라이스 |
| 데이터 스토어 | append-only JSONL(MVP) → SQLite → Postgres(스케일) |
| 진화 워커 언어 | TS(`packages/engine`) — 단일언어 단순성 |
| 모노레포 | npm workspaces + `scripts/gate.mjs` |
| 첫 실증 언어쌍 | **목표어=영어(en) / UI=한국어(ko)** — 효능 입증 후 Phase 4 에서 2번째 쌍 |

## 4. 데이터 소유·이동성 (규칙 6)

- 학습자는 `/api/account/export` 로 자기 데이터 전량(이벤트·상태) 내보내기, `/api/account/delete` 로 영구 삭제.
- 개방 데이터셋은 **동의된 익명 데이터만**, 재식별 불가 처리 후 별도 릴리스([`privacy-consent-open-data`]).

> 확정되면 `docs/design/` 에 세부 설계(데이터 모델 ERD·API 명세·어댑터 인터페이스)를 추가한다.
