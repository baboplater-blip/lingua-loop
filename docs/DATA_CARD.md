# 데이터 카드 (Data Card)

> 규칙 14·17: **카드 없이 데이터/모델 공개 금지.** 무엇을 수집하고, 어떻게 익명화하며, 한계·재식별 위험·라이선스가 무엇인지 투명하게 밝힙니다. SSOT: [`privacy-consent-open-data`](../.claude/skills/privacy-consent-open-data/SKILL.md).

## 1. 무엇을 수집하나

LinguaLoop의 유일한 원천 데이터는 **append-only 학습 이벤트 로그**입니다(규칙 5). 모든 상태(숙달도·능력·복습 스케줄)는 이 로그를 **리플레이해 파생**되며, 로그 외에 별도 원천은 없습니다.

이벤트 한 건의 스키마(`schemaVersion: 1`):

| 필드 | 예시 | 설명 |
|---|---|---|
| `eventId` | `evt_00001a` | 단조 증가 식별자(재시작 후 충돌 방지) |
| `ts` | ISO-8601 | 발생 시각 |
| `learnerRef` | `web-a1b2c3` | **가명(pseudonym)** — 실명·이메일 아님 |
| `type` | `item.response`·`tutor.turn` | 이벤트 종류 |
| `kc` | `["kc.en.present_be"]` | 관련 지식요소 |
| `itemId` | `en.be.am` | 문항 식별자 |
| `payload` | `{correct, grade}` / `{role, text}` | 종류별 세부(응답 정오·튜터 발화 등) |
| `consent` | `learn` | 이 이벤트의 사용 동의 범위 |

## 2. 개인정보·가명성

- **PII를 요구하지 않습니다.** 기본 웹앱은 `learnerRef`로 브라우저 로컬에서 생성한 랜덤 가명만 씁니다(이름·이메일·위치 없음).
- 튜터 대화(`tutor.turn`)는 학습자가 **직접 입력한 자유 텍스트**를 담을 수 있으므로 잠재적으로 개인 언급이 섞일 수 있습니다. 이 텍스트는 학습자 소유이며(규칙 6), 개방 데이터셋에는 **원문 그대로 포함하지 않습니다**.

## 3. 학습자 소유·통제 (규칙 6)

- **내보내기**: `GET /account/export?learner=<ref>` — 내 모든 이벤트를 JSON으로.
- **삭제**: `DELETE /account?learner=<ref>` — 메모리 + 영속 파일까지 **영구 삭제**(되돌릴 수 없음). 재시작해도 되살아나지 않습니다.
- 영속 저장은 운영자 디스크의 `data/events/<가명>.jsonl` 한 파일이며, 기본 `.gitignore`되어 커밋되지 않습니다.

## 4. 개방 데이터셋 정책

집계·연구용 개방 데이터셋을 배포할 경우 **모두 충족한 뒤에만** 공개합니다. 이 파이프라인은 `core/open-dataset.ts`로 **코드화**되어 있고, 운영자 도구 `npm run dataset:export`가 실행합니다(레드팀 실패 시 파일을 쓰지 않음):

1. **동의 필터** — `consent`가 `learn+improve+open`인 이벤트만(옵트인). 학습신호 이벤트만 선별.
2. **익명화** — 가명 재발급(원 가명과 링크 절단), 자유 텍스트 스크럽(text·message·note·reason 제거), 시각 일 단위 coarsen.
3. **재식별 레드팀 통과** — **k-익명성**: 준식별자(대략 프로파일: KC 수·이벤트 수·시작 월) 그룹이 k(기본 3) 미만이면 해당 학습자를 **억제**하고 재검증. 통과분만 릴리스. `/release --datasets`가 자동 실행.
4. **데이터 카드 자동 동봉** — `dataset:export`가 데이터셋(`*.jsonl`)과 함께 카드(`*-card.md`, `core/datasetCard`)를 생성합니다. 요약·스키마·익명화·k값·라이선스를 담아 **카드 없이 재배포 금지**를 코드로 강제.
5. **데이터 라이선스 명시**(§6).

> 원 이벤트 로그(운영자 보유분)는 **개방 데이터셋이 아닙니다.** 개방셋은 위 파이프라인을 거친 파생물입니다.

## 5. 한계·위험

- 자유 텍스트(튜터 대화)는 자동 익명화의 한계가 있어 개방셋에서 기본 제외/강한 마스킹 대상입니다.
- 소수 언어·소규모 코호트는 희소성으로 재식별 위험이 상대적으로 높아 집계 임계값(k-익명성)을 상향합니다.
- 로그는 append-only라 "수정"이 불가하며, 정정은 정정 이벤트를 **추가**하는 방식입니다(이력 무결성).

## 6. 라이선스

- **개방 데이터셋**(배포 시): **CC-BY-4.0 + 동의 조건** — §4의 익명화·재식별 레드팀 통과분만.
- 콘텐츠(언어팩)는 CC-BY-4.0, 코어 코드는 MIT로 **분리** 명시([`../LICENSES.md`](../LICENSES.md)).
- 라이선스는 동의 범위와 정합해야 하며, 동의하지 않은 데이터는 어떤 라이선스로도 배포하지 않습니다.
